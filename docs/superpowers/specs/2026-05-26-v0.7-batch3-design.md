# v0.7 Batch 3: Image/File Messages, Read Receipts, Message Recall

**Date:** 2026-05-26
**Version Target:** v0.7
**Batch:** 3 of 3

## Overview

Three features completing the v0.7 core functionality:
1. Image/file message support
2. Message read/unread status
3. Message recall (within 2 minutes)

---

## 1. Image/File Messages

### Approach
- `MsgTypeImage(2)` and `MsgTypeFile(3)` already existed in model
- Handler already accepted them in `handleChatMessage`
- File content stored as base64 in `Content` field via `FileContent` DTO
- Max file size: 5MB (client-side check)
- Received files saved to `DATA/<qq>/recv/` directory

### DTO
```go
FileContent{Filename, Size, Data}
```

### Client Commands
- `/sendimg <filepath>` — send image
- `/sendfile <filepath>` — send file

### Receiver Behavior
- Decodes base64 content and saves to `DATA/<qq>/recv/<fromQQ>_<filename>`
- Displays: `[Image] filename.jpg (1234 bytes)` or `[File] doc.pdf (5678 bytes)`

---

## 2. Message Read/Unread

### Data Model
- `Message.ReadAt *time.Time` — null = unread, set when read

### Service
- `MarkRead(messageID int64)` — sets `read_at` to current time (only if null)

### Message Type
- `MsgTypeReadReceipt = 420` — sent by receiver after displaying a message

### Client Behavior
- Auto-sends `MsgTypeReadReceipt` after displaying any received text/image/file message
- Uses same `AckRequest{MessageID}` payload as delivered ACK

---

## 3. Message Recall

### Data Model
- `Message.IsRecalled bool` — default false

### Service
- `RecallMessage(qq, messageID int64)` — validates:
  1. Message exists
  2. Caller is the sender
  3. Within 2 minutes of creation
  4. Not already recalled

### Message Types
| Constant | Value | Purpose |
|----------|-------|---------|
| `MsgTypeRecall` | 430 | Recall request |
| `MsgTypeRecallNotify` | 431 | Notify recipients of recall |

### DTOs
```go
RecallRequest{MessageID}
RecallNotify{MessageID, FromQQ, GroupID}
```

### History Filtering
- `GetHistoryWithTarget` and `GetGroupHistory` filter `is_recalled = false`
- Recalled messages disappear from history queries

### Client
- `/recall <message_id>` command
- Displays recall notification when receiving `MsgTypeRecallNotify`
