## 1. 发送前置校验（BUG-007）

### Problem
`/to` 一个不存在的用户后发消息，消息被存入数据库但目标永远收不到。

### Solution
在 `handleChatMessage` 中，调用 `svc.HandleMessage` 存储消息之前，先调用 `svc.GetUserByUID(msg.ToUID)` 校验目标用户是否存在。

### Changes
- **File:** `internal/handler/ws.go` — `handleChatMessage`
- **Logic:**
  ```
  if _, err := h.svc.GetUserByUID(msg.ToUID); err != nil {
      // 返回错误 ServerAck，不存储消息
      return
  }
  // 继续原有流程：存储 + 发送
  ```
- **Error response:** `MsgTypeServerAck` with content "user not found"

### Verification
- `/to nonexistent_user` → 发消息 → 收到 "user not found" 错误
- `/to existing_user` → 发消息 → 正常发送

## 2. 好友分组管理重做

### Problem
当前 `/movefriend <qq> <group>` 会自动创建不存在的分组，不符合 IM 常规设计。

### Solution
引入 `friend_groups` 表，分组必须先创建再使用。

### Database
新建 `friend_groups` 表：
```go
type FriendGroup struct {
    ID        uint      `gorm:"primaryKey;autoIncrement"`
    UID       string    `gorm:"index;not null"`
    GroupName string    `gorm:"not null"`
    CreatedAt time.Time
}
// Unique index: (uid, group_name)
```

### Service Layer (chat.go)
新增方法：
- `CreateFriendGroup(uid, name string) error` — 创建分组，重复则报错
- `DeleteFriendGroup(uid, name string) error` — 删除分组，组内有好友则报错
- `GetFriendGroups(uid string) ([]string, error)` — 已有，保持不变

修改方法：
- `MoveFriendGroup(uid, friendQQNumber int64, groupName string) error` — 移动前先查 `friend_groups` 表，分组不存在则返回 `ErrGroupNotFound`

### Handler Layer (ws.go)
新增消息类型：
- `MsgTypeFriendCreateGroup MessageType = 309`
- `MsgTypeFriendDeleteGroup MessageType = 310`

新增路由和 handler：
- `handleFriendCreateGroup` — 调用 `CreateFriendGroup`
- `handleFriendDeleteGroup` — 调用 `DeleteFriendGroup`

### Client (client/main.go)
新增命令：
- `/creategroup <name>` — 发送 `MsgTypeFriendCreateGroup`
- `/delgroup <name>` — 发送 `MsgTypeFriendDeleteGroup`
- `/help` 增加新命令说明

### Verification
- `/creategroup 家人` → 成功
- `/movefriend 10001 家人` → 成功
- `/movefriend 10001 不存在的组` → 报错
- `/delgroup 家人`（组内有好友）→ 报错
- `/delgroup 空组` → 成功

## 3. 客户端内 `/login`

### Problem
登录只在 client 启动时执行一次，登录失败后只能重启 client。

### Solution
在客户端增加 `/login <uid> <password>` 命令，通过已有 WebSocket 连接发送登录请求。

### Changes
- **File:** `cmd/client/main.go` — `handleCommand`
- **Command:** `/login <uid> <password>`
- **Logic:**
  1. 发送 `MsgTypeLogin` 消息（payload: `LoginRequest{UID, Password}`）
  2. 收到 `MsgTypeLoginAck` 后：
     - `code == 0`: 更新 `currentUser = uid`, `myQQNumber = resp.QQNumber`，打印登录成功
     - `code != 0`: 打印错误信息，不关闭连接

### Server
无需改动。`handleLogin` 已支持密码登录流程。

### Verification
- 启动 client 不登录 → `/login alice password` → 登录成功
- `/login alice wrongpassword` → 报错，连接保持

## 4. `/whoami` 用户信息

### Problem
用户无法查看当前登录账号信息。

### Solution
纯客户端本地实现，使用已持有的 `currentUser` 和 `myQQNumber` 变量。

### Changes
- **File:** `cmd/client/main.go` — `handleCommand`
- **Command:** `/whoami`
- **Logic:**
  ```
  if myQQNumber == 0 {
      print "not logged in"
  } else {
      print "UID: {currentUser}, QQ: {myQQNumber}"
  }
  ```

### Verification
- 未登录 → `/whoami` → "not logged in"
- 登录后 → `/whoami` → 显示 UID 和 QQ 号
