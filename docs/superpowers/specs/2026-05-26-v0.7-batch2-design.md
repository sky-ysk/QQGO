# v0.7 Batch 2: Change Password & Blacklist

**Date:** 2026-05-26
**Version Target:** v0.7
**Batch:** 2 of 3

## Overview

Two core features:
1. Change password with token invalidation
2. User blacklist (block/unblock/list + message rejection)

---

## 1. Change Password

### Service Layer
- `ChangePassword(qq int64, oldPassword, newPassword string) (string, error)`
- bcrypt compare old password → bcrypt hash new password → update `users.password_hash` + `users.token`
- Returns new token (old sessions are invalidated)

### Message Types
| Constant | Value | Purpose |
|----------|-------|---------|
| `MsgTypeChangePassword` | 400 | Change password request |
| `MsgTypeChangePasswordAck` | 401 | Change password response |

### DTOs
```go
ChangePasswordRequest{OldPassword, NewPassword}
ChangePasswordResponse{Code, Message, Token}
```

### Handler
- `handleChangePassword` validates login, calls service, sends ack with new token

### Client
- `/changepw <old_password> <new_password>` command
- On success: saves new token to local file

### Tests
- `TestChangePassword`: register → change password → login with new OK → login with old fails
- `TestChangePasswordWrongOld`: wrong old password → ErrAuthFailed

## 2. Blacklist

### Data Model
```go
Blacklist{ID, QQ, BlockedQQ, CreatedAt}
```
Unique index on `(qq, blocked_qq)`.

### Service Layer
- `BlockUser(qq, blockedQQ int64) error` — check target exists, not self, not already blocked
- `UnblockUser(qq, blockedQQ int64) error` — delete record
- `IsBlocked(qq, blockedQQ int64) bool` — check if qq has blocked blockedQQ
- `GetBlacklist(qq int64) ([]BlockedUserInfo, error)` — list all blocked users

### Message Types
| Constant | Value | Purpose |
|----------|-------|---------|
| `MsgTypeBlockUser` | 410 | Block user |
| `MsgTypeUnblockUser` | 411 | Unblock user |
| `MsgTypeBlacklist` | 412 | List blocked users |

### Message Rejection
In `handleChatMessage` (handler/ws.go), before non-friend check:
- If `IsBlocked(toQQ, fromQQ)` → reject with "you are blocked by the recipient"

### Client
- `/block <qq>` — block a user
- `/unblock <qq>` — unblock a user
- `/blacklist` — list blocked users
- Updated `/help`

### Tests
- `TestBlockUser`: block → IsBlocked true → block again fails
- `TestBlockSelf`: block self → error
- `TestUnblockUser`: block → unblock → IsBlocked false → unblock again fails
- `TestGetBlacklist`: block 2 users → list returns both
