# v0.3.1 测试方法

**版本：** v0.3.1
**分支：** `feature/v0.3.1-fix-and-polish`
**日期：** 2026-05-22

---

## 准备工作

```bash
# 1. 确认在 feature/v0.3.1-fix-and-polish 分支
git branch --show-current

# 2. 删除旧数据库（FriendGroup 新表需要 AutoMigrate）
rm -f qqgo.db

# 3. 启动服务端
go run ./cmd/server

# 4. 打开 2 个终端，分别启动客户端
# 终端 A:
go run ./cmd/client alice

# 终端 B:
go run ./cmd/client bob
```

### 2. 好友分组管理重做

**前置条件：** alice 和 bob 已是好友

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 2.1 | alice: `/creategroup 家人` | 成功 |
| 2.2 | alice: `/groups` | 列表包含"家人" |
| 2.3 | alice: `/creategroup 家人` | 报错 "group already exists" |
| 2.4 | alice: `/creategroup 我的好友` | 报错 "invalid group name" |
| 2.5 | alice: `/creategroup 待处理` | 报错 "invalid group name" |
| 2.6 | alice: `/movefriend <bob_qq> 同事` | 报错 "group not found" |

| 2.9 | alice: `/delgroup 同事`（组内有 bob） | 报错 "group is not empty" |
| 2.10 | alice: `/movefriend <bob_qq> 我的好友` → `/delgroup 同事` | 成功删除"同事"分组 |
| 2.11 | alice: `/delgroup 我的好友` | 报错 "cannot delete default group" |
| 2.12 | alice: `/creategroup 空组` → `/groups` | "空组"出现在列表中（空分组可见） |

### 3. 客户端内 /login

| 步骤 | 操作 | 预期结果 |
